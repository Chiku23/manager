Project Alpha ALF-124

Login page throws 500 on invalid credentials

Urgent In Progress Bug Sprint 4

Description

When a user enters an invalid email/password combination on the login page and submits the form, the server responds with an unhandled 500 Internal Server Error instead of the expected 401 Unauthorized.

Steps to reproduce:

  1. Navigate to /login
  2. Enter an email that does not exist in the database
  3. Enter any password and click "Sign In"

Expected: Return 401 with a user-friendly error message.

Actual: Server throws 500, error is logged to console.
